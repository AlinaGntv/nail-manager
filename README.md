# Telegram Nail AI Manager

Telegram AI-менеджер для мастера маникюра Анны.

Бот работает через FastAPI webhook, отвечает клиентам через OpenAI Responses API, собирает имя, телефон, услугу, желаемые дату и время, сохраняет готовую запись в SQLite и Google Sheets, а затем отправляет мастеру уведомление в Telegram.

## Настройка

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Создайте `.env`:

```bash
copy .env.example .env
```

3. Заполните реальные ключи:

```env
OPENAI_API_KEY=your-polza-api-key
OPENAI_MODEL=deepseek/deepseek-v4-flash
OPENAI_BASE_URL=https://polza.ai/api/v1
BOT_TOKEN=123456789:telegram-bot-token
MANAGER_CHAT_ID=123456789
GOOGLE_SERVICE_ACCOUNT_FILE=./secrets/google-service-account.json
GOOGLE_SHEET_ID=your-google-sheet-id
```

> Бот использует Polza (OpenAI-compatible) вместо OpenAI напрямую. Ключ Polza может начинаться не с `sk-`, это нормально.

4. При необходимости измените бизнес-настройки без правки кода:

```env
MASTER_NAME=Анна
WORK_HOURS=пн-сб 10:00-20:00
STUDIO_ADDRESS=г. Москва, ул. Примерная, 10, кабинет 5
MANICURE_PRICE=от 1500 руб.
PEDICURE_PRICE=от 2200 руб.
GEL_POLISH_PRICE=от 2000 руб.
NAIL_EXTENSION_PRICE=от 3500 руб.
```

## Webhook Telegram

Telegram должен видеть публичный HTTPS URL. Для локального теста используйте ngrok или cloudflared.

```bash
curl -X POST "https://api.telegram.org/bot<BOT_TOKEN>/setWebhook" \
  -d "url=https://example.com/telegram/webhook" \
  -d "secret_token=change-me"
```

`secret_token` должен совпадать с `TELEGRAM_WEBHOOK_SECRET`.

## Планы: расписание и запись по слотам

### Что нужно реализовать

Система управления расписанием мастера, чтобы бот мог проверять свободные слоты и не записывать клиента на занятое время.

### Архитектура

Два слоя расписания:

1. **Базовое расписание на неделю** — настраивается один раз. Пример: пн-сб 10:00-20:00, вс выходной. Хранится в SQLite.
2. **Исключения по датам** — конкретный день перекрывает базу: выходной (28.06), сокращённый день (01.07 12-18), перенос. Хранится в отдельной таблице.

### Как бот проверяет слот

- Клиент пишет "суббота 10:00".
- Бот смотрит: есть ли исключение на эту дату? Если нет — берёт из базового расписания.
- Проверяет, нет ли уже записи на это время (с учётом длительности услуги + буфер).
- Если свободно — принимает заявку. Если нет — предлагает другое время.

### Команды для мастера (Анны) в Telegram

- `/schedule` — показать расписание на текущую неделю
- `/block 28.06` — заблокировать дату (выходной)
- `/unblock 28.06` — убрать блокировку
- `/sethours 01.07 12-18` — задать часы на конкретный день
- `/available 28.06` — показать свободные слоты на дату

### Таблицы в SQLite

- `schedule` — базовое расписание (день недели + время начала/конца)
- `schedule_exceptions` — исключения (конкретная дата + время или "выходной")
- `bookings` — записи клиентов (дата, время, услуга, длительность)

### Дополнительно

- Праздничные дни РФ автоматически выходные (1-8 января, 23 февраля, 8 марта, 1 мая, 9 мая, 12 июня, 4 ноября).
- Буфер между записями — 30 минут по умолчанию.
- Длительность услуги считается автоматически: маникюр ~1.5ч, педикюр ~2ч, наращивание ~3ч.
- Интеграция: бот перед записью проверяет расписание и сообщает клиенту свободные слоты.
